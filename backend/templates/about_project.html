Prepared by: Waqar Maqbool Wani (23045136010) & Inshpreet Kour Mehta (23045136011)

View on GitHub

Introduction

This project is an AI-powered data analytics platform that enables users to analyze datasets and extract insights through a natural language interface. Users can upload datasets (CSV, Excel, JSON), and the system automatically generates descriptive statistics, visualizations, and actionable insights. The platform features an integrated NLP chatbot powered by the OpenRouter API (OpenAI-compatible, using the Qwen model), allowing users to interact with their data using plain English queries. The chatbot responds with relevant summaries, charts, or dashboard updates, making data analysis accessible to non-technical users.

Key Features

  • Data Upload & Processing (CSV, Excel, JSON formats) with automatic schema detection and type inference.
  • Automated Analysis: Summary statistics, visualizations, pattern and anomaly detection using Pandas, Numpy, and Seaborn.
  • AI-Powered Data Summarization: Generates concise, human-readable summaries of datasets using the OpenRouter API.
  • Advanced Filtering: Apply complex, multi-condition filters to datasets via natural language or structured queries.
  • NLP Chatbot: Natural language queries via OpenRouter API (Qwen model), supporting context-aware data exploration and manipulation.
  • Interactive Dashboard: Real-time filtering, sorting, and export options using Plotly, with support for 2D and 3D charts.
  • Secure API Integration: All AI and data operations are performed securely with local deployment and environment-based API key management.
  • User Reviews: Collect and display user feedback within the platform, stored in a secure database.
  • Extensible Architecture: Modular backend and frontend design for easy integration of new analytics or visualization modules.

Use Cases

  • Healthcare: Analyze patient data to detect disease trends, predict risks, and automate compliance reporting using advanced filtering and AI-powered summarization.
  • Education: Monitor student performance, identify at-risk learners, and generate automated insights for curriculum improvement and resource allocation.
  • Business: Visualize sales data, customer demographics, manage inventory efficiently, and perform churn analysis or market segmentation using interactive dashboards.
  • Finance: Perform fraud detection, risk assessment, and portfolio analysis on large financial datasets with automated anomaly detection and natural language querying.
  • IoT & Sensor Data: Aggregate and analyze real-time sensor data streams, detect anomalies, and generate actionable alerts using AI-driven analytics.
  • HR Analytics: Analyze employee data for attrition prediction, diversity metrics, and performance trends using advanced filtering and visualization tools.
  • Research & Academia: Rapidly explore and summarize experimental or survey data, generate publication-ready charts, and automate data cleaning workflows.
  • Government & Public Sector: Analyze census, public health, or transportation data to inform policy decisions and improve service delivery with secure, local data processing.

Problem Formulation

Modern organizations generate vast and heterogeneous datasets from multiple sources (CSV, Excel, JSON, sensors, databases), making it challenging to extract actionable insights. Non-technical users often lack the expertise to perform advanced analytics or data cleaning. Traditional BI tools (e.g., Tableau, Power BI) require manual configuration, lack natural language interfaces, and are not always suitable for real-time or ad hoc analysis. Furthermore, using general-purpose cloud AI models for sensitive data raises privacy and compliance concerns. There is a critical need for a secure, explainable, and user-friendly analytics platform that enables real-time, natural language-driven data exploration and insight generation for all users, regardless of technical background.

Proposed Solution

  • AI-Powered Natural Language Querying: Enables users to interact with datasets using plain English, powered by OpenRouter API (Qwen model) for context-aware data analysis and filtering.
  • Automated Data Analysis & Visualization: Instantly generates descriptive statistics, advanced visualizations (2D/3D), and actionable insights using Pandas, Numpy, Plotly, and Seaborn.
  • Secure Local Data Processing: All data operations and AI queries are performed on the user's local environment, ensuring privacy and compliance for sensitive datasets.
  • Modular & Extensible Architecture: Backend and frontend are designed for easy integration of new analytics modules, visualization types, or external APIs.
  • Interactive Dashboards & User Feedback: Real-time, user-friendly dashboards with export options and built-in user review system to drive continuous improvement.

Objectives

  • Enable Seamless Data Upload and Processing
  • Provide Automated Insights and Visualizations
  • Integrate NLP-Based Querying Capabilities (OpenRouter API, Qwen)
  • Ensure Data Privacy and Security
  • Support Domain-Specific Use Cases
  • Promote Scalability and Flexibility

Methodology/Planning of Work

  • Frontend: Flask (Jinja2 templates), HTML, CSS, JavaScript. Uses AJAX for asynchronous data upload, dashboard updates, and chatbot communication. Responsive UI for data preview, charting, and chatbot interaction.
  • Backend: Flask REST API with modular blueprints for NLP, reviews, and data processing. Handles authentication, session management, and secure API key usage.
  • Data Processing: Pandas and Numpy for schema inference, data cleaning, normalization, and advanced multi-condition filtering. Supports CSV, Excel, and JSON formats.
  • Analytics Engine: Plotly and Seaborn for generating interactive 2D/3D charts, anomaly detection, and exportable visualizations. Supports real-time dashboard updates.
  • NLP Chatbot: OpenRouter API (Qwen model) for context-aware natural language queries, filter extraction, and AI-powered data summarization. Integrates securely with backend for data-driven responses.
  • Database: SQLite (default, via SQLAlchemy ORM) for user reviews and session data, with optional support for PostgreSQL and MongoDB for scalability and backup.
  • Frontend: Flask (Jinja2 templates), HTML, CSS, JavaScript - File upload, dashboards, chatbot UI
  • Backend: Flask - API handling, authentication, data processing
  • Data Processing: Pandas, Numpy - Cleaning, preprocessing
  • Analytics Engine: Plotly, Seaborn - Visualizations, anomaly detection
  • NLP Chatbot: OpenRouter API (Qwen model) - Query interpretation and data interaction
  • Database: SQLite (default), with support for PostgreSQL and MongoDB - Storage and backup
